# Contributing to OrgCanvas

Thanks for your interest in contributing to OrgCanvas! We welcome contributions of all kinds: bug reports, feature requests, documentation improvements, and pull requests.

## How to contribute

1. Fork the repository and create a feature branch from `master`:
   - `git checkout -b feat/short-description`
2. Make changes in a branch and keep commits small and focused.
3. Run lint and tests locally before opening a PR:
   - `npm install`
   - `npm run lint`
   - (add tests if you add logic)
4. Push your branch and open a Pull Request against `master` with a clear description.

## Code style
- Use TypeScript and follow existing patterns in the repo.
- Keep UI changes accessible and follow a11y best practices.
- Re-run `npm run lint` and `npm run build` before opening PR.

## Issues
- Report bugs with steps to reproduce and a minimal example if possible.
- Feature requests should include rationale and basic mockups if helpful.

## PR Checklist
- [ ] My change addresses a single issue or feature.
- [ ] I added tests where appropriate.
- [ ] I updated documentation when needed (README, SECURITY, etc.).
- [ ] Lint passes: `npm run lint`.
